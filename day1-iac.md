## What IaC is and the problem it solves

Infrastructure as Code (IaC) is used to tools like make software delivery faster and easier by treating almost all operations be it servers, databases, or documentation as code. Simply, instead of developers and operations teams manually setting up servers in cloud or running commands manually, they just write code to manage infrastructure. The benefits include: manage everything like software, tasks automated instead of manual and faster deployment with minimal errors

---

## Difference Between Declarative and Imperative code

An IaC tool that uses Imperative language instructs the system step-by-step what to do and must track what already exists. For example, you have 10 existing servers and wants to add up to 15, the earlier code will not be helpful and rerunning the code with 15 will deploy new servers giving a total of 25 so one needs to be aware of what is already deployed and write a new code to add the extra servers. In contrast, declarative language just declares the end result and figures on it own on how to achieve that. On the earlier example, declarative code is aware of the existing servers and just add what is needed. 

---

## Why Terraform?

Terraform is worth learning as an Infrastructure as Code (IaC) tool in DevOps because it automates infrastructure, significantly reducing manual effort and errors. It uses a declarative approach, allowing you to define the desired end state while Terraform handles how to achieve it. Additionally, it makes it easy to track changes and collaborate since infrastructure can be stored and version-controlled in Git.
